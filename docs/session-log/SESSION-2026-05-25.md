## Key decisions (this session)
- **Chat MVP = BYOK + TEXT CHAT + TRANSIENT VISION.** Vision: image input is **not stored** server-side (downscale client-side → base64 → relay → discard); only the analysis + filename persist. Defer image/video **generation** until a media-storage strategy.
- **Governance — three buckets:** UI = freely reversible (go fast); **data model = the one-way door (always gated: propose→approve→ADR)**; the **capability/relay layer** (caching, web-search, tool-calls, MCP) = ONE consistent "power-strip" plug-in pattern + ADR, never per-feature hacks.
- **Data model (researched 2 rounds + owner-confirmed):** a **normalized per-message "message tree"** (ChatGPT/Claude's parent-pointer branching, in Vercel's table style) — NOT a JSON blob. One row per message (cheap appends for long chats, scales to many users); `parentId` so compare/regenerate are additive; flexible JSON `settings` bag; per-message metrics (model/tokens/cost/time); transient-vision attachments via `parts` + optional `storageKey`. → **ADR-0013**.

- **Backend on the Vercel AI SDK** (`ai` + OpenRouter provider) — streaming + provider handling + path to resumable streams (vs hand-rolled SSE).
- **Prompt caching ON by default** (new vs the local app) — via the one request-builder; off-switch in chat settings.

## Key decisions / direction (this session — major strategic expansion)
*(Captured in detail in personal memory; summarized here for the repo handoff.)*
- **Vision expanded beyond Phase 1:** evolve into a broad, ChatGPT/Claude-shaped **cloud multimodal + workflow AI workspace** — "Higgsfield + a real chat brain (memory + RAG)." Roadmap: generative modalities (chat/image/audio/video) → workflows → user-extensible platform (users add their own APIs + MCPs).
- **Architecture principle — "API-everything / serverless relay":** add capabilities as external API integrations; never run heavy compute ourselves. In scope: transcription, ElevenLabs-style speech-to-speech (async API calls).
- **The ceiling (hard lines, until big MRR + stability):** no VM-backed autonomous agents; no realtime/persistent-connection features (live canvas, voice-mode); no enterprise; cloud-only. → now public in `docs/vision.md`.
- **Business model:** flat **~$5/mo infra-only**, **never resell AI credits** (always BYOK); MIT open-source self-host; data-ownership ladder (hosted / hosted+BYO-storage / self-host); **BYO object storage = the sweet spot** (cost + privacy + content-liability offload); team/workspace **usage governance** = metering + budget allocation on a *shared* key (NOT credit resale).
- **Content liability:** BYOK offloads cost + the generation *act*, NOT *hosting* liability (media in our R2 on our domain = we're the host). Keep media off Vercel, serve via presigned R2 URLs → Vercel/Neon stay low-risk. No illegal content (CSAM scanning/reporting); legal NSFW left to providers/unmarketed but still safety-gated.
- **Competitive benchmark = Higgsfield** (credit-based multi-model video/image gen + cinematic controls + avatars; ~15M users, $1.3B). We build ~80% **+ BYOK + chat/memory/RAG**. Feasibility risk: which marquee models are BYOK-API-accessible (Sora/Veo may not be; video needs fal/Replicate → more keys = more friction).
- **Positioning: learning-first, dual-audience** — non-technical = learning platform (guided videos/courses + learn-and-experiment-live); technical = BYOK tool. Learning = primary hook + the low-CAC acquisition engine; in-app `/learn` = a core front-door surface.

## Decisions made (with rationale)
- **Greenfield rebuild, feature-by-feature** — clean, understood codebase + learning value. → [ADR-0006](../adr/0006-rebuild-from-scratch.md)
- **Vercel (serverless) hosting** — Next-native, ~$0, easy CI/CD + previews. → [ADR-0002](../adr/0002-host-on-vercel-serverless.md)
- **Cloudflare R2 for media (off Vercel)** — Vercel AUP prohibits the content; R2 has no egress fees. → [ADR-0003](../adr/0003-cloudflare-r2-for-media.md)
- **Store keys not URLs** + **direct-to-storage uploads** — durable refs; dodge the 4.5 MB limit. → [ADR-0004](../adr/0004-store-storage-keys-not-urls.md), [ADR-0005](../adr/0005-direct-to-storage-uploads.md)
- **Stack: Next 16.2.6, plain CSS, npm** — match version-accurate docs, keep it simple. → [ADR-0007](../adr/0007-stack-next16-plain-css.md)
- **MIT, public, open source.**
- **Multi-agent: lighter start** — agents as *available helpers*, no hard-routing mandate. **Why:** routing isn't mechanically enforceable, and a rigid mandate adds friction + is premature before any feature code exists.
- **Commit identity = GitHub no-reply email** — keep the real email off a public repo.
- **Pre-flight audit before the first commit** — it's going public; do it properly.
- **Go anonymous before publishing** — renamed the GitHub account to `cosmicbubble898` ("Cosmic Bubble"), scrubbed the real name from LICENSE/README/PRD, and authored commits as "Cosmic Bubble" with the masked no-reply email.
